# Сравнение трёх продуктов: IM ITV vs SecurOS Dispatcher vs наш WEB

**Дата:** 2026-05-13
**Источники анализа:**
- IM ITV — исходники в `Сompetitors/IM ITV/IM/` (C++/MFC + C#/WPF, тесная связь с Intellect).
- SecurOS Dispatcher v11 — официальные User Guide и Administration Guide (PDF, май 2025), см. `competitor-securos-dispatcher.md`.
- Наш WEB — текущее состояние пакета DSL v1 (`README.md`, `dsl-v1-draft.md`, `configurator-web/`, `runner-web/`).

---

## 0. Профили продуктов

| Параметр | IM ITV (legacy) | SecurOS Dispatcher v11 | Наш WEB (DSL v1) |
|---|---|---|---|
| **Производитель** | ITV / AxxonSoft (старый стек) | Intelligent Security Systems (ISS) | AxxonSoft (новый стек) |
| **Тип клиента** | Толстый: WinForms + WPF | **Web (браузер)** | **Web (React/TS)** + iOS/Android в перспективе |
| **Платформа сервера** | C# WCF + SQL Server, IIDK к Intellect | Внутри SecurOS, **PostgreSQL only** | Go + PostgreSQL (отдельный репо) |
| **Связь с VMS** | Intellect: IIDK + COM | SecurOS: Видеосервер + WebConnect + Карта | Открыто, через сервер (планируется Axxon One и др.) |
| **Зрелость** | Production, legacy (на замену) | Production v11, активно развивается | **Pre-alpha**: спецификация + 2 SPA-демо |
| **Подход к UI** | Imperative (код XAML) | Server-side rendering UI элементов | **Server-Driven UI (декларативный JSON)** |
| **Язык конфигурации сценария** | Параметры в `Msg` (Intellect-формат), `goto`/`macro` | Визуальный graph-builder Да/Нет | **Декларативный JSON + JSONLogic (16 операторов whitelist)** |
| **БД** | SQL Server | PostgreSQL only | PostgreSQL (JSONB для script и result) |
| **Версионирование сценариев** | Нет (изменения «вступают в силу немедленно») | Нет (правка регламента сразу действует) | **Да: `(scenarioGuid, version)`** — старые инциденты на своей версии |
| **Мульти-тенантность** | Нет | Не описана в гайдах | **Заложена** (`customer_guid`) |
| **Локализация** | ru, kk, en (фрагментарно) | ru, en (smen на лету) | Inline-строки, плановая `labelKey`-механика (v2) |

## 3. Стратегические выводы по сравнению

### 3.1. Что у нас лучше — нужно сохранить и подчеркнуть

0. **VMS-агностичность (главный стратегический эксклюзив).** Адаптеры в Go-бэкенде на Axxon Next/One, SecurOS, Milestone, Genetec, ONVIF, MQTT/Kafka/REST. Конкуренты намертво привязаны: IM ITV — к Intellect, SecurOS Dispatcher — к SecurOS VMS. Мы единственные на рынке работаем поверх **любой** инфраструктуры (см. §1.18).
1. **Декларативный DSL + JSONLogic** — невероятная гибкость для сложных сценариев, недостижимая в SecurOS (Да/Нет) и IM ITV (`Msg`-параметры).
2. **Версионирование сценариев `(guid, version)`** — никто не умеет; критично для безопасных правок и аудита.
3. **Snapshot-based result** — самодостаточные инциденты в архиве, не зависящие от текущей версии сценария.
4. **`history[].userId` per step** — поддержка многопользовательских сценариев из коробки.
5. **Open Schema + Examples** — конфиги в Git, code review, regression-набор.
6. **Кросс-платформенный runner** (Web + iOS + Android) — полностью отсутствует у конкурентов.
7. **Server-Driven UI** — обновления без перекомпиляции мобильных клиентов.
8. **Live preview, undo/redo, dual view (flow/table), diagnostics panel** — современный UX редактора.

### 3.2. Что у SecurOS лучше — обязательно реализовать

1. **Полная иерархия Датчик→Локация→Строение→ПУ.**
2. **5 ролей с понятным распределением полномочий.**
3. **Авто-эскалация по таймауту между ролями.**
4. **Подсистема перерывов операторов** (целостная).
5. **Плановые работы** + временное отключение датчиков.
6. **Справочник обслуживающих компаний и сотрудников.**
7. **Уровни тревоги Локаций.**
8. **Дашборд с 4 ключевыми диаграммами.**
9. **Полный аудит действий** с фильтрами и UI.
10. **Шаблоны фильтров для архива.**
11. **Звуковые сигналы** для инцидентов и уведомлений.
12. **Прикрепление файлов** к инциденту.
13. **Темы и языки на лету.**
14. **Группировка инцидентов в ленте** по Локации с реакцией на всю группу.
15. **Принудительное завершение сеанса** другого оператора (СОДЦ).
16. **Составной инцидент** (несколько условий с разных датчиков).
17. **Восстановление архивного инцидента** (одиночное и массовое).

### 3.3. Что у IM ITV лучше — учесть

1. **Гибкие per-user права** (булевы флаги) — у SecurOS только в рамках роли. Нам стоит дать **гибридную модель**: роли + опциональные кастомные пермишны.
2. **Настройка горячих клавиш операторов** (`HotKeys`).
3. **Цвета приоритетов** настраиваются админом (per-event-type).
4. **Группировка инцидентов настраивается** (по чему группировать).
5. **Вызов макросов Intellect** (мы уже заложили `callMacro`).
6. **Многоязычность с казахским** в комплекте (расширить наш список).

### 3.4. Что слабо везде — наш шанс выделиться

0. **VMS-агностичность** — оба конкурента жёстко привязаны (IM ITV → Intellect, SecurOS → SecurOS VMS); наш Go-бэкенд может работать поверх любой VMS через adapter-слой. **Главный стратегический эксклюзив.**
1. **AD/LDAP/SSO/SAML/OAuth2** — у всех болевая точка.
2. **GIS-карты с реальной геопривязкой** (OSM/Яндекс/Google), а не растровая «карта SecurOS».
3. **Версионирование** (только мы).
4. **Открытое API для конфигурации** через Git (только мы).
5. **Полноценный BI-интерфейс** с экспортом в Excel/CSV/PowerBI.
6. **Webhooks** на события инцидентов.
7. **Mobile-first для ГБР** (нативные приложения, push-уведомления).
8. **Antivirus / DLP-проверка** прикреплённых файлов.
9. **Шаблоны регламентов** (Markdown, изображения, ссылки, таблицы — не только UL/LI).
10. **Шаблоны сценариев** (сценарий как «функция», переиспользование) — отчасти заложено в `examples/` библиотечных.
11. **Аналитика по сценариям** (конверсия по веткам, узкие места) — никто не делает.
12. **Симулятор сценариев** (run-from-step, mock-data) — у нас уже есть в раннере.

## 5. Сильнейшая короткая формула

**SecurOS Dispatcher** = «зрелый организационный продукт с богатыми справочниками, ролями и аудитом, **намертво вшитый внутрь SecurOS VMS** и с примитивным движком сценариев».

**IM ITV** = «гибкий технический продукт с произвольными ветвлениями и Intellect-макросами, **намертво вшитый внутрь Intellect**, без современного UX, дашбордов и операционных подсистем».

**Наш WEB + Go** = «современная open-friendly DSL-платформа с непревзойдённой гибкостью движка, версионированием, кросс-платформенностью и **VMS-агностичной архитектурой** (работает поверх любой VMS через adapter-слой), но пока без операционной обвязки».

**Наша цель:** взять *операционную зрелость* SecurOS, *гибкость движка* IM ITV/JSONLogic, **сохранить наши эксклюзивы** (версионирование, multi-platform, GIS, SSO, BI, schema-first, **VMS-agnostic**) — и получить продукт, который **закрывает SecurOS на тендерах с любой VMS, кроме самого SecurOS, и одновременно превосходит его операционно**.

См. конкретные шаги реализации в `web-feature-recommendations.md`.
